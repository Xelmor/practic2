Этап 1. Прототип CLI-инструмента
На первом этапе создан консольный интерфейс, который принимает входные параметры и проверяет корректность введенных данных.
Поддерживаемые параметры:
•	--package-name – имя анализируемого пакета
•	--repo-url – URL или локальный путь к репозиторию
•	--mode – режим работы: local или remote
•	--version – версия пакета
•	--filter – фильтрация зависимостей по подстроке
Пример запуска:
python main.py --package-name pandas --mode remote --version 2.3.2
Этап 2. Извлечение прямых зависимостей
На втором этапе реализовано получение зависимостей двумя способами:
1) Режим local
2) Режим remote
Получаются:
•	метаданные пакета,
•	список его зависимостей (поле requires_dist),
•	при необходимости выполняется фильтрация по подстроке --filter.
Этап 3. Построение графа зависимостей
На заключительном этапе реализована визуализация зависимостей с помощью библиотеки graphviz.
